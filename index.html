My final year project is a blockchain based dApp (decentralised application), built on the Ethereum platform. It is an application to model a basic supply chain. The functionality will enable a Producer to create an item, which will be made available for purchase by and delivery to a Middleman, who in turn can pay the Producer and make the item available for purchase and delivery to an End User - who can pay the Middleman and consume the item, ending the chain.

All payments within the system will be made in Ether (the native crypto currency to Ethereum platform) - and I will be using the Goerli test net to provide "free" Ether for development purposes.


The blockchain platform was chosen as an alternative to a traditional database backend as by design this platform has in-built immutability and traceability characteristics - enabling the items lifecycle to be traced directly back to the source without any doubt upon the integrity of the journey.


The project is an end to end full stack application, making use of the following tools:

  • Ethereum

    (Solidity, Remix, Metamask, Ganache, Infura)

  • Nethereum

    (C# .Net Core framework for interfacing with Ethereum smart contracts)

  • .Net Core

    (Mid-tier server side application to handle business logic of the application and manage service classes, MSTest framework for unit testing)

  • Blazor

    (WASM Web Assembly front end framework for C# for the UI)

  • Azure

    (App Service for deployment & hosting, Pipelines for CI/CD)
